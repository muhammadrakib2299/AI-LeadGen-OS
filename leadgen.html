What it is

AI LeadGen OS turns a natural-language question — "restaurants in Paris with vegan options," "B2B SaaS in Berlin," "law firms in Amsterdam" — into a verified, deduplicated, GDPR-compliant list of business leads with name, website, email, phone, and provenance per field.

It also accepts a CSV of company names you already have ("here are 500 prospects, find me their websites and decision-maker emails") and returns the enriched file in about ten minutes.

The system is built for a solo operator or small team running a B2B outreach business in the EU/UK: every cost is capped per job, every fetch is logged for audit, every PII column is encrypted at rest, and a one-flag "Compliant Mode" restricts the pipeline to Tier-1 official APIs only.

Project status: 100% of the original scope shipped. 334 passing tests. Multi-tenant SaaS with Stripe billing, team seats, CRM connectors (HubSpot + Pipedrive), export destinations (CSV + S3 + Google Sheets), AI Ask Mode (filter + vector RAG), API key rotation, scheduled retention/reverify cron. Ready to deploy.

Who it's for

🎯 Solo B2B operator

Selling outbound services in the EU/UK. You need 100–500 verified leads a week, can't afford an Apollo seat, and refuse to ship anything LinkedIn-scraped to your clients.

🏢 Outreach agency

Running campaigns for 3–10 clients in parallel. You want each client's data isolated (multi-tenant), need CRM push to wherever they live (HubSpot or Pipedrive), and need to bill them via Stripe without writing the integration yourself.

📊 Sales-ops at a small SaaS

Want to enrich your existing CRM with EU contacts you already have permission to email, plus discover new ICPs. The bulk-CSV mode is your friend.

Why now

Three forces are converging right now:

  • Apollo / ZoomInfo / Lusha all rely on LinkedIn scraping. EU enforcement is tightening (CNIL fines, ICO scrutiny). Buyers want a clean alternative.
  • Cheap LLMs (Haiku-class) make NL → structured query parsing a $0.0001 op — you can charge for it.
  • pgvector + cheap embeddings mean "Ask Mode" RAG is a one-day build, not a quarter-long ML project.

  • arq cron in the same worker means we don't pay for a separate scheduler service.

AI model overview

Three model roles, each chosen for cost vs. quality fit:

Role Model Why this one Cost shape
Query parser (NL → structured search) Claude Haiku 4.5 Fast, cheap, JSON-mode reliable. Escalates to Sonnet only for low-confidence parses. ~$0.0001 per query
Contact extractor (HTML → email/phone/social) Claude Haiku 4.5 Regex pre-filters most cases; Haiku catches the formatted-image / obfuscated ones. ~$0.0003 per page
Premium escalation (ambiguous queries) Claude Sonnet 4.6 Triggered when Haiku's confidence < 0.5 AND an entity_type was detected. ~$0.003 per escalation
Ask Mode v1 (NL question → SQL filter) Claude Haiku 4.5 The model only sees the schema + question — never row PII. Whitelisted filter fields. ~$0.0001 per question
Ask Mode v2 (vector similarity) OpenAI text-embedding-3-small Cheapest dense embedding at 1536 dim with strong B2B relevance. Only non-PII text embedded. $0.02 / 1M tokens (≈ $0.00002 per entity)
GDPR & embeddings. Only the joined string "name | city | country | category" is sent to OpenAI. Email, phone, and address never leave the platform — there's an explicit regression-guard test for this.

How lead generation works

The pipeline is one process composed of named stages. Each stage writes to Postgres so the next stage can resume from any point if a worker crashes or a job is re-tried.

Stages

1. Validate

NL query → structured spec. Rule parser first, LLM second, escalate-on-low-confidence third.

2. Discover

SmartRouter calls adapters in priority order. Cache hit short-circuits the network. Per-domain rate limit honored.

3. Crawl

Visit homepage + contact + about (multilingual paths). robots.txt enforced. Retry-After cooldown set on 429/503.

4. Extract

Regex finds the easy emails/phones; Haiku is invoked only for pages where regex came up empty.

5. Verify

Email syntax + MX (always); Hunter (only top-quality candidates). Phone format + country code. URL liveness.

6. Score

Quality 0–100 from completeness, freshness, source trust. Threshold below 60 → review queue.

7. Dedupe

Fuzzy match on normalized name + domain + address. Lower-quality row marked duplicate_of.

8. Persist

Entity + per-field provenance + RawFetch audit row. PII columns encrypted on the way in.

9. Export

Signed CSV URL by default; on-demand push to S3 / Sheets / HubSpot / Pipedrive.

Where money is spent (and capped)

  • Per-domain rate limit caps how fast we hammer a small site.
  • Per-tenant + per-job budget cap — when cost_usd approaches budget_cap_usd, the runner stops enqueuing new fetches and marks the job budget_exceeded.
  • Tiered LLM routing — Haiku is the default; Sonnet only when Haiku is unsure.
  • Short-lived response cache — repeat queries within ~1h are served from Redis without an upstream call.
  • Email-pattern cache (Postgres) — once we know {firstname}.{lastname}@acme.com works for one Acme contact, we don't re-spend on Hunter for the next.
